November 2020 update

Welcome to the updated Alexa Skills Toolkit for November 2020 (version 2.2). This update introduces a number of new features, enhancements, and bug fixes.

What's new?

Dynamic content in welcome screen

We enhanced the Welcome screen on toolkit launch, to include quick info that is helpful to skill developers.

Developer blog

This section provides a quick glance on the latest Alexa developer blog posts.

Feature updates

This section provides a quick glance on the recent releases and feature updates in Alexa Skills Kit.

Documentation tree - What's new

Since the dynamic content gets updated frequently, and should be available to skill developers on demand, we reorganized the What's new link under Documentation tree to open the Welcome screen as and when needed.

Region support in local debugging launch configuration

We recently released an update to local debugging that allows you to configure the endpoint to be used. This toolkit version supports that feature through debug configurations for local debugging. You can find more information regarding this on our official docs here.

AWS default region support in create Alexa-hosted skill view

Skill developers can select the AWS region to be used as the default skill endpoint, in the create Alexa-hosted skills workflow. More information on Alexa-hosted skills region support can be checked here.

Bug fixes

  • 42: Update apl-suggester version, that updates validations and suggestions for APL documents.
  • 43: Fix typo for Portuguese locale in views.
  • 56: Fix git working directory for skill clone.
  • 57: Fix toolkit to work with VSCode latest release 1.51.1.

Documentation

We found some toolkit users had issues with using the skill clone/deploy feature. To help future developers facing similar issues, we added information about the git functionality in the toolkit. You can read about that in our FAQs section here.
